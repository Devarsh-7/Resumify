# Project Architecture: Resumify AI (Stitch UI + Antigravity Backend)

## 1. UI Implementation Rules (Strict Boundaries)
- NEVER generate new UI components from scratch.
- ALWAYS reference the components in `/src/components/stitch-ui`.
- If a new UI element is needed, ask me to create it in Google Stitch first.
- Treat the Stitch UI directory as an immutable source of truth for all design, spacing, layouts, and Tailwind classes.

## 2. Backend & Logic (Antigravity's Domain)
- Your primary role is to build and maintain the backend logic, databases, API routes, and frontend state management.
- You are responsible for connecting the existing Stitch UI components to the backend.
- You may add event handlers (e.g., `onClick`, `onSubmit`), state hooks, and API fetch calls to the Stitch components, but you must strictly preserve their existing DOM structure and Tailwind CSS classes.

## 3. Tech Stack
- **Frontend:** React.js / Tailwind CSS (Generated by Google Stitch)
- **Backend:** Node.js with Express
- **Database:** MongoDB (Mongoose for ODM)
- **State Management:** React Context API / Custom Hooks

## 4. Standard Feature Workflow
When asked to build a new feature, follow this exact sequence:
1. Locate the relevant UI component in `/src/components/stitch-ui`.
2. Build the corresponding backend MongoDB models and Express API routes.
3. Wire the Stitch frontend component to the new API route.
4. Verify functionality without breaking the original visual design.
